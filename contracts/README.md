# AegisEscrow Demo Contract

This directory contains a small Foundry project for a demo-native-token escrow contract that can be used by the Aegis backend during development and integration demos.

## Versions

- Solidity: `0.8.33`
- OpenZeppelin Contracts: `5.x`
- Foundry: latest stable release recommended

## Contract

`AegisEscrow` supports:

- native token deposits keyed by a backend-provided `escrowId`
- admin-controlled release to the recipient
- admin-controlled refund to the payer
- enum-based escrow status tracking
- events for `Deposited`, `Released`, and `Refunded`

The contract uses OpenZeppelin `Ownable` for admin control and `ReentrancyGuard` for the release/refund flow.

## Setup

Install Foundry if it is not already available:

```bash
curl -L https://foundry.paradigm.xyz | bash
foundryup
```

Install dependencies from the `contracts/` directory:

```bash
cd contracts
forge install OpenZeppelin/openzeppelin-contracts --no-commit
forge install foundry-rs/forge-std --no-commit
```

## Test

Run the full test suite:

```bash
cd contracts
forge test -vvv
```

## Build

Compile the contract:

```bash
cd contracts
forge build
```

## Deploy

Set the deployer key and desired admin account:

```bash
export PRIVATE_KEY=0x...
export AEGIS_ESCROW_ADMIN=0xYourAdminAddress
```

Deploy with Foundry:

```bash
cd contracts
forge script script/DeployAegisEscrow.s.sol:DeployAegisEscrow \
  --rpc-url $RPC_URL \
  --broadcast
```

## Demo Integration Notes

- `escrowId` is intended to be generated by the backend so off-chain records can map directly to on-chain escrow state.
- `deposit` is payable and creates the escrow record in a single transaction.
- `release` and `refund` are intentionally admin-only so the backend can orchestrate demo flows without additional on-chain approval logic.
- `statusOf` and `getEscrow` are suitable for simple backend polling and demo dashboards.
